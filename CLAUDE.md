# CLAUDE.md

This file is intentionally minimal. **All project knowledge lives in [`docs/`](./docs/README.md)** — read it there.

## How to use this repo as Claude Code

1. **Always consult `docs/` first.** It is the single source of truth for this monorepo: architecture, lifecycle, layout, events, assets, conventions, build pipeline, deployment, versioning rules, and the game-creation workflow.
2. **Index of docs:** [`docs/README.md`](./docs/README.md). It links every topic file plus [`docs/PIXIJS-docs.txt`](./docs/PIXIJS-docs.txt), the full PixiJS API reference for everything the engine doesn't wrap.
3. **Never add new project knowledge to this file.** When you learn or decide something worth persisting (a convention, a lifecycle rule, a build-pipeline detail, a deployment note), put it in the appropriate file under `docs/` — extend an existing doc or add a new one and link it from `docs/README.md`. Keep `CLAUDE.md` empty of substantive content.
4. **Small repo-level reminders only.** Anything that belongs here is a one-liner pointing at `docs/`, not the knowledge itself.
